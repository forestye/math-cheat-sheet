# 📐 高阶程序员数学武器库·生产指南 (Production Guide for Math Cheat Sheets)

## 1. 核心哲学 (Core Philosophy)
我们的目标不是“压缩教科书”，而是**“编写工程索引”**。
* **Target Audience**: 资深工程师、架构师、算法研究员。
* **Key Value**: “遇到工程瓶颈 $\to$ 查表 $\to$ 找到数学降维打击工具”。
* **Tone**: 极简、硬核、直觉优先。拒绝废话，拒绝纯理论推导。

---

## 2. 视觉规范 (Visual Standards)

所有 Cheat Sheet 必须基于统一的 LaTeX 模板架构。

### 2.1 页面布局 (Layout)
* **Class**: `\documentclass[8pt, landscape, a4paper]{extarticle}` (必须用 8pt 以保证信息密度)。
* **Geometry**: `\usepackage[margin=0.8cm, top=1cm, bottom=1.3cm]{geometry}` (严格统一边距)。
* **Grid**: **4 栏布局** (`\begin{multicols*}{4}`)。
* **No Page Number**: `\pagestyle{empty}`。

### 2.2 配色系统 (Color System)
每张 Cheat Sheet 必须有一个**主色调 (Theme Color)**，用于区分不同学科，但在同一张表中保持统一。

| 角色 | 颜色定义 | 说明 |
| :--- | :--- | :--- |
| **Header Blue** | `RGB{R, G, B}` | **主色调**。用于标题、列表点、边框。随学科变化。 |
| **Section 1** | 同主色调 | 用于普通模块边框。 |
| **Nav Color** | `RGB{211, 84, 0}` (橙) | **固定**。仅用于“场景导航”模块。 |
| **Intuition** | `RGB{41, 128, 185}` (蓝) | **固定**。仅用于右下角“核心直觉”模块（冷色思考）。 |
| **Accent** | `RGB{192, 57, 43}` (红) | **固定**。仅用于极其重要的公式（如 SVD、贝叶斯）。 |
| **Divider** | `RGB{220, 220, 220}` | **固定**。用于分割线，极淡灰色。 |

**学科建议色：**
* 🟢 **线性代数**: Green / Teal (基石)
* 🔵 **微积分/复变**: Blue (流动/变化)
* 🟣 **概率统计**: Purple (神秘/不确定性)
* 🟠 **图论**: Orange (连接/网络)
* 🔴 **抽象代数**: Red (规则/结构)
* 🟤 **凸优化**: Brown/Bronze (土地/代价)

### 2.3 样式组件 (Components)
* **Box**: 圆角 `arc=2mm`，内边距 `left=4pt`，行距 `linespread{1.1}`。
* **Separator**: 必须使用自定义的 `\sepline` (强制换行+画线)，禁止直接用 `\hrulefill`。
* **Formula**:
    * 核心公式：**独占一行** (`$$...$$`) 以示强调。
    * 次要公式：**行内紧凑** (`$...$`) 以省空间。

## 3. 内容架构 (Content Framework)

每张 Cheat Sheet 必须包含以下 **6 大核心模块**，分布在 4 栏中：

### **第 1 栏：入口与公理 (The Hook)**
1.  **[置顶] 🗺️ 场景导航 (Use Cases)**
    * **颜色**: `navcolor` (橙色)。
    * **格式**: `工程痛点 -> 数学工具`。
    * *例*：“系统发散 -> 极点分析”。
2.  **基础定义 (Foundations)**
    * 该学科的“原子”概念（如复数定义、向量空间公理）。
    * 必须包含核心定义的几何意义。

### **第 2 栏：核心算力 (The Engine)**
3.  **核心定理/公式 (Core Tools)**
    * 该学科最硬核的计算工具（如积分定理、SVD、特征值）。
    * **排版要求**: 公式要大，解释要短。
    * **技巧**: 必须包含“进阶技巧”或“注意事项”。

### **第 3 栏：工程落地 (The Bridge)**
4.  **变换/映射 (Transforms)** (可选)
    * 连接理论与应用的桥梁（如傅里叶、拉普拉斯）。
5.  **工程应用 (Engineering)**
    * **必须关联**: 信号处理、控制、物理、AI、网络等具体领域。
    * *拒绝*：“应用广泛”这种空话。*要求*：“用于计算机翼升力”。
6.  **算法与代码 (Code/Algo)**
    * **语言**: Python (Numpy/Scipy) 或 C++。
    * **内容**: 核心数据结构、关键函数名、数值稳定性技巧（如 `logsumexp`, `fftshift`）。

### **第 4 栏：升维与直觉 (The Soul)**
7.  **上帝视角/高阶理论 (Endgame)**
    * 该学科的“天花板”或与其他学科的交叉点（如黎曼曲面、流形、谱图论）。
8.  **[置底] 💡 核心直觉 (Intuition)**
    * **颜色**: `intuitioncolor` (蓝色)。
    * **必选**: **TikZ 矢量配图**（一图胜千言）。
    * **文案**: 哲学升华，用一句话总结该学科的本质（如“复数是旋转的虫洞”）。
    * **对齐**: 使用 `\vspace*{\fill}` 确保沉底对齐。

## 4. 编写检查清单 (Checklist)

在生成代码前，请自我核对：

* [ ] **信息密度**: 是否有废话？（删除所有“我们知道”、“众所周知”）。
* [ ] **工程映射**: 是否至少有 3 个具体的工程场景（如“推荐系统”、“死锁检测”）？
* [ ] **代码实战**: 代码片段是否是真实的库函数（如 `np.linalg.svd`）？
* [ ] **公式美学**: 核心公式是否独占一行？变量是否清晰？
* [ ] **视觉平衡**: 4 栏底部是否对齐？（误差 < 1行）。
* [ ] **TikZ 插图**: 核心直觉部分是否有图？图是否使用了 `color=...` 正确语法？
